System Architecture

MIDGUARD sits as a transparent security gateway between users and AI agents, enforcing a 5-phase pipeline on every request and every response.

👤
Human User
Browser / API client
POST /v1/gateway
→
🛡️ MIDGUARD
Phase 1 Auth + Rate Limit
  • API Key validation via PostgreSQL
  • Token Bucket rate limiter (Redis)
→
Phase 2 Policy Engine
  • YAML rule evaluation
  • Keyword + action + network rules
→
Phase 3 Threat Detection
  • Prompt injection (transformer)
  • Jailbreak patterns
  • PII scanner (spaCy NER)
  • Toxicity (LLM Guard)
→
Phase 4 Enforcement
  • Combines Phase 2+3 signals
  • ALLOW / BLOCK / QUARANTINE
  • Writes audit log
↓ if ALLOW
Phase 5 Output Filter
  • PII redaction in AI response
  • Hallucination check
  • Output toxicity scan
